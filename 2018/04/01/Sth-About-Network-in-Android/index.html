Something About Network in Android

Android
  • Android
  • 网络请求

TIP: 网络分层模型，HTTPS和HTTP, Retrofit，OKHttp

网络分层模型

TCP/IP模型分为4层：应用层、传输层、网络层、网络接口层
我们一般涉及到的也就是应用层上，这里也就是OSI分层模型中的应用、表示、会话层

HTTPS和HTTP

 我们通常所说的HTTP协议其实是应用层的一个传输协议，这个不用再多作解释了，而HTTPS协议就是为了解决HTTP协议的不安全性，在HTTP协议下加了SSL（OSI模型的表示层上），HTTPS比HTTP增加一次握手，这个握手的过程
1.客户端发起HTTPS请求
2.服务端将自己的CA证书给客服端，CA证书中有很多信息，包括公钥
3.客户端验证证书安全性
4.将客户端私钥用公钥加密传递给服务端
5.服务端用服务端私钥解密客户端传递过来的私钥（非对称加密）
6.将数据用客户端私钥加密进行数据传递（对称加密）
而另外说的HTTP的三次握手是TCP协议中的客户端和服务端进行的建立连接的过程
HTTPS的加密过程是在传输层进行的，所以用charles抓包工具抓包看到的还是明文数据
 注：android 7.0默认不信任用户导入的CA证书了，所以访问自定义CA证书服务器需要进行一些处理，通过TrustManagerFactoty其中的一些方法，加入自定义CA证书

OKHttp框架

1.初始化OKHttpClient
 其实就是进行一些配置
2.构建Request
 进行url的拼接，请求方法等赋值
3.获取Call对象
 创建RealCall调用
4.发送同步/异步请求
 同步请求，利用client.dispatcher().executed(this) 来进行实际执行,getResponseWithInterceptorChain()函数获取HTTP返回结果，异步请求，利用内部维护的线程池进行网络请求
拦截器是okHttp中最核心的东西
 责任链模式：多个对象有机会处理请求，从而避免了请求的发送者和接收者之间的耦合，将这些对象连成一条链，并沿着这条链传递请求，直到有对象处理它。
okHttp中的拦截器用到的就是责任链模式。
 拦截器可以进行请求重试重定向，构造请求和用户友好请求响应，缓存，建立连接，发送请求读取响应 
private Response getResponseWithInterceptorChain() throws IOException {
    // Build a full stack of interceptors.
       List<Interceptor> interceptors = new ArrayList<>();
    interceptors.addAll(client.interceptors());
    interceptors.add(retryAndFollowUpInterceptor);
    interceptors.add(new BridgeInterceptor(client.cookieJar()));
    interceptors.add(new CacheInterceptor(client.internalCache()));
    interceptors.add(new ConnectInterceptor(client));
    if (!retryAndFollowUpInterceptor.isForWebSocket()) {
          interceptors.addAll(client.networkInterceptors());
    }
    interceptors.add(new CallServerInterceptor(
        retryAndFollowUpInterceptor.isForWebSocket()));
    Interceptor.Chain chain = new RealInterceptorChain(
    interceptors, null, null, null, 0, originalRequest);
    return chain.proceed(originalRequest);
}
使用OKHttp进行网络请求可以简化网络请求书写，内部实现了缓存，请求压缩，管理线程池等部分

Retrofit框架

 那么在介绍了OKHttp框架之后，其实已经完全可以满足一个客户端进行网络请求的需求了，那么为什么还会有Retrofit框架的使用呢？
Retrofit对OKHttp进一步封装，使网络请求更加简洁，方便
 Retrofit内部使用了大量的设计模式（外观模式、代理模式、工厂模式、单例模式、建造者模式等）使客户端进行网络请求解耦合，将业务逻辑与实现细节分离，方便了上层开发者，具体过程
1.通过解析网络请求接口的注解配置网络请求参数
2.通过动态代理生成网络请求对象
3.通过网络请求适配器将网络请求对象进行平台适配，目前大部分使用Rxjava适配
4.通过网络请求执行器发送网络请求（默认OKHttp）
5.通过数据转换器解析服务器返回的数据（如Gson）
6.通过回调执行器切换线程
 第一步创建Retrofit实例 
private static Retrofit createRetrofit() {
    Retrofit.Builder retrofitBuilder = new Retrofit.Builder();
    retrofit = retrofitBuilder.baseUrl(BASE_URL)
            .client(okHttpClient)
            .addCallAdapterFactory(RxJavaCallAdapterFactory.createWithScheduler(Schedulers.io()))
            .addConverterFactory(GsonConverterFactory.create())
            .build();
    return retrofit;
}
就是对Retrofit内的成员变量进行配置，Retrofit中的一些比较重要的成员变量
serviceMethod：包含所有网络请求信息的对象
baseUrl：网络请求的url地址
callFactory：网络请求工厂
adapterFactories：网络请求适配器工厂的集合
converterFactories：数据转换器工厂的集合
callbackExecutor：回调方法执行器

public <T> T create(final Class<T> service) {
    return (T) Proxy.newProxyInstance(service.getClassLoader(), new Class<?>[] { service },
        new InvocationHandler() {
              @Override 
              public Object invoke(Object proxy, Method method, Object... args)throws Throwable {
                  //第一点
                ServiceMethod serviceMethod = loadServiceMethod(method);
                //第二点
                OkHttpCall okHttpCall = new OkHttpCall<>(serviceMethod, args);
                //第三点
                return serviceMethod.callAdapter.adapt(okHttpCall);
              }
    });
  }
大写的动态代理，过动态代理创建网络请求接口的实例，当我们调用接口方法时会转发到InvocationHandler的invoke方法中去实现
下面我们看看ServiceMethod这个类是干什么的 

result = new ServiceMethod.Builder(this, method).build();
public Builder(Retrofit retrofit, Method method) {
    this.retrofit = retrofit;
    this.method = method;
    this.methodAnnotations = method.getAnnotations();
    this.parameterTypes = method.getGenericParameterTypes();
    this.parameterAnnotationsArray = method.getParameterAnnotations();
}

public ServiceMethod build() {
    callAdapter = createCallAdapter();
    responseType = callAdapter.responseType();
    responseConverter = createResponseConverter();
    // 主要是解析获取Http请求的方法
    for (Annotation annotation : methodAnnotations) {
        parseMethodAnnotation(annotation);
    }

    int parameterCount = parameterAnnotationsArray.length;
    parameterHandlers = new ParameterHandler<?>[parameterCount];
    for (int p = 0; p < parameterCount; p++) {
        Type parameterType = parameterTypes[p];
        Annotation[] parameterAnnotations = parameterAnnotationsArray[p];
        parameterHandlers[p] = parseParameter(p, parameterType, parameterAnnotations);
    }
    return new ServiceMethod<>(this);
}

获取网络请求接口方法里的注释，获取网络请求接口方法里的参数类型，获取网络请求接口方法里的注解内容，封装到ServiceMethod的成员变量中，ServiceMethod里面包含了访问网络的所有基本信息
然后传入serviceMethod创建OKHttpCall，这个才是真正网络请求的执行，内部使用OKHttp的进行网络请求
最后调用CallAdapter的adapt方法，它其实是使用了装饰者模式，内部真正发起请求的其实是OKHtttpCall，而这层装饰的主要作用就是切换线程，都知道OKHttp的一个缺点是，它没有帮我们进行线程的切换，而在异步请求的回调方法中我们需要手动切换到主线程中去修改UI，这个CallAdapter我们可以给它传入RxjavaCallAdapterFactory，轻松使用线程调度符进行线程的切换
最后强行进行一波总结：
Retrofit将Http请求抽象成接口
在接口里用注解描述和配置 网络请求参数
用动态代理的方式，动态将网络请求接口的注解解析成HTTP请求
最后执行HTTP请求
